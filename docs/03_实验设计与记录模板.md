# 实验设计与记录模板

> 根据 PDF，至少需要 3 个清晰的 hypothesis-driven experiments。下面给的是低风险、可复现、适合当前修复仓库的默认方案。

---

## 1. 实验统一原则

### 固定不变项
除非实验专门研究该变量，否则统一固定：
- 数据：`train-mini.json` + `dev-v1.1.json`
- 预处理参数：`para_limit=400`, `ques_limit=50`
- batch size：`8`
- seed：`42`
- loss：`qa_nll`
- 保存方式：每组实验用独立 `save_dir` / `log_dir`
- 评估指标：`loss`, `F1`, `EM`

### 最低记录要求
每个实验记录：
- 假设（Hypothesis）
- 自变量（Independent Variable）
- 保持不变项（Controls）
- 配置命令 / notebook 参数
- 最终 train loss / dev loss / dev F1 / dev EM
- 对结果的解释

---

## 2. Experiment 1 — Optimizer

### 假设
在当前修复后的 QANet 上，`adam` 相比 `sgd` 会更快降低训练损失，并可能在相同步数下获得更高的 dev F1，但也可能引入更强的波动。

### 对比项
- Baseline: `optimizer_name="sgd"`
- Variant: `optimizer_name="adam"`

### 保持不变
- `scheduler_name="step"`
- `lr_step_size=10000`
- `lr_gamma=0.5`
- `num_steps=200` 或相同步数

### 建议保存目录
- SGD: `save_dir="exp_opt_sgd"`, `log_dir="log_opt_sgd"`
- Adam: `save_dir="exp_opt_adam"`, `log_dir="log_opt_adam"`

### 分析点
- 哪个优化器更快降低训练 loss？
- 哪个优化器 dev F1/EM 更好？
- 指标是否稳定？是否出现过拟合迹象？

---

## 3. Experiment 2 — Normalization

### 假设
`layer_norm` 与 `group_norm` 会影响训练稳定性和泛化，其中 `layer_norm` 可能更贴近原始 QANet 设计，而 `group_norm` 可能在小 batch 情况下表现出不同的稳定性。

### 对比项
- Baseline: `norm_name="layer_norm"`
- Variant: `norm_name="group_norm"`, `norm_groups=8`

### 保持不变
- optimizer 固定为 `sgd` 或 `adam`（选一个并保持一致）
- scheduler 固定为 `step`
- 其余训练参数一致

### 建议保存目录
- LayerNorm: `exp_norm_layer`
- GroupNorm: `exp_norm_group`

### 分析点
- 哪种 norm 更稳定？
- 哪种 norm 在 dev F1/EM 上更好？
- 小 batch 下 normalization 差异是否明显？

---

## 4. Experiment 3 — Scheduler 或 Regularization

可二选一，建议优先做 scheduler。

### 方案 A：Scheduler
#### 假设
不同学习率调度策略会影响收敛节奏和最终泛化，其中 `cosine` 可能带来更平滑的优化过程。

#### 对比项
- Baseline: `scheduler_name="step"`
- Variant: `scheduler_name="cosine"`

#### 保持不变
- optimizer 固定
- num_steps 相同
- 其余参数相同

#### 建议保存目录
- Step: `exp_sched_step`
- Cosine: `exp_sched_cosine`

#### 分析点
- loss 曲线是否更平滑？
- dev F1/EM 是否提升？
- 后期训练是否更稳定？

### 方案 B：Dropout
#### 假设
适度 dropout 有助于泛化，但过高 dropout 会抑制学习能力。

#### 对比项
- Baseline: `dropout=0.1`
- Variant: `dropout=0.2` 或 `dropout=0.05`

#### 分析点
- 训练 loss 与 dev loss 的平衡
- 是否出现 underfitting / overfitting

## 5. 推荐实验运行模板

```python
from TrainTools.train import train

results = train(
    train_npz="_data/train.npz",
    dev_npz="_data/dev.npz",

    char_emb_json="_data/char_emb.json",
    train_eval_json="_data/train_eval.json",
    dev_eval_json="_data/dev_eval.json",
    save_dir="YOUR_SAVE_DIR",
    log_dir="YOUR_LOG_DIR",
    num_steps=200,
    checkpoint=50,
    batch_size=8,
    seed=42,
    optimizer_name="sgd",
    scheduler_name="step",
    lr_step_size=10000,
    lr_gamma=0.5,
    loss_name="qa_nll",
    norm_name="layer_norm",
)

print(results)
```

## 7. 报告写法模板

每个实验建议按如下结构写：

### Research Question
我们希望研究 ______ 是否会影响 ______。

### Hypothesis
我们假设 ______，原因是 ______。

### Setup
- 数据：
- 固定参数：
- 变化参数：
- 评估指标：

### Results
插入表格或曲线。

### Analysis
- 结果是否支持假设？
- 观察到的差异可能来自什么机制？
- 该实验的局限性是什么？
